# Security policy

## Учебный раннер

Python Path выполняет только короткий код для учебных задач. Перед запуском код проверяется по AST, импорт ограничивается allowlist, а файлы доступны только через временную виртуальную папку (`open`/безопасный `pathlib`). Выполнение идёт отдельным Python-процессом с ограниченным набором встроенных функций и завершается по таймауту. Системные, процессные и произвольные сетевые операции закрыты.

Это снижает риск для локального тренажёра, но не заменяет инфраструктурную изоляцию. Не размещайте текущую версию как публичный сервис, в котором чужие пользователи могут исполнять произвольный код.

Для публичного режима потребуются как минимум:

- отдельные контейнеры или microVM на каждое выполнение;
- лимиты CPU, памяти, файловой системы, сети и числа процессов;
- очереди заданий, rate limiting и аудит;
- отдельное хранилище с аутентификацией пользователей.

## Сообщить о проблеме

Не публикуйте детали потенциальной уязвимости в Issue. Создайте приватное security advisory в GitHub или свяжитесь с владельцем репозитория напрямую.
