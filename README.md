## Зачем этот проект

Python Path делает обучение практичным и последовательным. Ученик проходит материал маленькими шагами, получает мгновенную обратную связь по ответам и коду, возвращается к ошибочным заданиям и видит свой прогресс в XP, звёздах и серии занятий.

## Быстрый старт

Требуются Python 3.11+ и [uv](https://docs.astral.sh/uv/).

```powershell
git clone https://github.com/gon7187/python-path.git
cd python-path
uv sync --extra dev
uv run uvicorn app.main:app --reload
```

Откройте [http://127.0.0.1:8000](http://127.0.0.1:8000). Прогресс сохраняется локально в `python_path.db`, который исключён из Git.

Чтобы открыть приложение с телефона в той же Wi-Fi-сети, запустите сервер так:

```powershell
uv run uvicorn app.main:app --host 0.0.0.0 --port 8000
```

## Программа курса

Курс следует от базового синтаксиса к реальным инструментам разработки:

1. Первые программы, типы, условия, циклы, функции и коллекции.
2. Строки, модули, файлы, JSON/CSV, исключения и ООП.
3. Типы, стандартная библиотека, регулярные выражения, тесты и Git.
4. CLI, окружения, HTTP/API, SQL, параллельность и async.
5. Четыре выпускных мини-проекта: задачи, заметки, API-клиент и релиз приложения.

Перед основным курсом есть «Спокойный старт» из 19 микроуроков: `print`, строки, переменные, типы, `len`, f-строки, ошибки и повторение — без перегрузки несколькими новыми идеями сразу.

Полная последовательность из 32 разделов — в [карте курса](docs/CURRICULUM.md).

## Возможности

- Каждый урок содержит минимум четыре карточки теории, разбор примера и три задания.
- Перед каждым заданием показан план решения; подсказку можно открыть до ответа.
- Следующий урок открывается после успешного прохождения предыдущего.
- После каждого раздела открывается контрольная точка; для зачёта требуется 70% правильных ответов.
- Практика работает сериями по три шага: текущая тема, повторение ошибок, смешанное повторение или выбранный открытый раздел.
- Ошибки из практики автоматически попадают в отдельный мягкий режим повторения.
- Код проверяется в изолированном Python-процессе с AST-фильтром и двухсекундным лимитом.
- `input()` работает через безопасный симулятор: ответы вводятся под редактором, по одной строке для каждого вызова.
- Один пользователь, локальный SQLite: можно заниматься без регистрации и облака.

## Как устроена практика

Практика не подбрасывает случайные вопросы. Она собирает короткую серию из трёх шагов и предлагает четыре режима:

1. **Текущий шаг** — закрепить тему, которая сейчас открыта в маршруте.
2. **Ошибки** — вернуться к заданиям, в которых ответ был неверным.
3. **Смешанная серия** — повторить уже открытые темы в порядке «правило → короткий ответ → код».
4. **Выбранный раздел** — точечно потренировать доступную тему.

У каждого задания есть план решения, а после серии показывается результат и понятный следующий шаг.

## Личный прогресс и тесты

Прогресс хранится только в локальном `python_path.db`. Файл базы и его резервные копии не попадают в Git. Автотесты используют отдельную временную SQLite-базу, поэтому запуск `pytest` не должен менять ваш учебный прогресс.

## Разработка и проверки

```powershell
uv run ruff check app tests
uv run ruff format --check app tests
uv run pyright app tests
uv run pytest -q
```

В репозитории есть GitHub Actions, которые выполняют эти проверки на каждом push и pull request.

## Структура

```text

  content.py              # базовые уроки и объединение программы

```

## Безопасность учебного раннера

Встроенный редактор рассчитан на короткие учебные задачи: запрещены импорты и системные операции, решение запускается отдельным процессом и останавливается по таймауту. Это удобная защита для локального тренажёра, но **не** полноценный sandbox для публичного многоарендного сервиса. Перед публикацией в интернете нужен отдельный контейнерный раннер с лимитами CPU, памяти и сети. Детали — в [SECURITY.md](SECURITY.md).
