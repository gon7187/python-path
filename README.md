## Зачем этот проект

Python Path делает обучение практичным и последовательным. Ученик проходит материал маленькими шагами, получает мгновенную обратную связь по ответам и коду, возвращается к ошибочным заданиям и видит свой прогресс в XP, звёздах и серии занятий.

## Быстрый старт

Требуются Python 3.11+ и [uv](https://docs.astral.sh/uv/).

```powershell
git clone https://github.com/gon7187/python-path.git
cd python-path
uv sync --extra dev
uv run uvicorn app.main:app --reload
```

Откройте [http://127.0.0.1:8000](http://127.0.0.1:8000). Прогресс сохраняется локально в `python_path.db`, который исключён из Git.

Чтобы открыть приложение с телефона в той же Wi-Fi-сети, запустите сервер так:

```powershell
uv run uvicorn app.main:app --host 0.0.0.0 --port 8000
```

## Программа курса

Курс следует от базового синтаксиса к реальным инструментам разработки:

1. Первые программы, типы, условия, циклы, функции и коллекции.
2. Строки, модули, файлы, JSON/CSV, исключения и ООП.
3. Типы, стандартная библиотека, регулярные выражения, тесты и Git.
4. CLI, окружения, HTTP/API, SQL, параллельность и async.
5. Четыре выпускных мини-проекта: задачи, заметки, API-клиент и релиз приложения.

Перед основным курсом есть «Спокойный старт» из 20 микроуроков: `print`, строки, переменные, типы, `len`, f-строки, `input`, преобразования и строковые методы — без перегрузки несколькими новыми идеями сразу. Пороговые темы тоже раздроблены: отдельно изучаются `if`/`else`, цикл/накопитель, `while`/`break`, вызов/параметр/`return`, список/`append`, `split`, `sum`, словарь и множество.

Полная последовательность из 32 разделов — в [карте курса](docs/CURRICULUM.md). Правила, по которым материал строится для человека, а не для формального прохождения, закреплены в [библии обучения](docs/LEARNING_DESIGN.md).

## Возможности

- Каждый урок содержит подробную теорию, исполняемые примеры с выводом и переход от узнавания к самостоятельному коду.
- Используются выбор, свободное воспроизведение, Parsons-блоки, код и накопительные финальные задачи.
- Подсказки исчезают постепенно: направление → план → почти готовая форма.
- Следующий урок открывается после успешного прохождения предыдущего.
- Урок нельзя сдать, пропустив обязательный код; контрольная точка требует 70% и практические задания.
- Практика берёт только завершённые уроки, чередует форматы и распределяет повторения по интервалам 1/3/7/14/30 дней.
- Ошибка попадает в мягкий режим повторения и исчезает из слабых мест после двух последовательных верных решений.
- Код проверяется в изолированном Python-процессе с AST-фильтром и двухсекундным лимитом.
- `input()` работает через безопасный симулятор: ответы вводятся под редактором, по одной строке для каждого вызова.
- Вкладка «Песочница» даёт свободный редактор с историей запусков и консолью: можно экспериментировать без проверки задания.
- После сдачи урока и серии практики показывается реальный консольный вывод выполненного кода.
- В разделе «Проекты» есть шесть мини-инструментов: они открываются по конкретным prerequisite-урокам и проверяются на нескольких входных сценариях.
- Curriculum-linter проверяет метаданные, скрытые тесты и запрет незнакомых конструкций до объяснения.
- Один пользователь, локальный SQLite: можно заниматься без регистрации и облака.

## Как устроена практика

Практика не подбрасывает случайные вопросы. Она ранжирует уже изученное по ошибкам, сроку повторения и числу попыток, затем предлагает четыре режима:

1. **Последняя тема** — закрепить последний завершённый урок; будущие конструкции сюда не попадают.
2. **Ошибки** — вернуться к заданиям, в которых ответ был неверным.
3. **Смешанная серия** — чередовать выбор, воспроизведение, Parsons и код из разных изученных тем.
4. **Выбранный раздел** — тренировать все завершённые уроки раздела, а не только последний.

У каждого задания есть план решения, а после серии показывается результат и понятный следующий шаг.

## Личный прогресс и тесты

Прогресс хранится только в локальном `python_path.db`. Файл базы и его резервные копии не попадают в Git. Автотесты используют отдельную временную SQLite-базу, поэтому запуск `pytest` не должен менять ваш учебный прогресс.

## Разработка и проверки

```powershell
uv run ruff check app tests
uv run ruff format --check app tests
uv run pyright app tests
uv run pytest -q
```

В репозитории есть GitHub Actions, которые выполняют эти проверки на каждом push и pull request.

## Структура

```text
app/
  main.py                 # FastAPI-приложение и JSON API
  content.py              # базовые уроки и объединение программы
  foundation_expansion.py # плавные микроуроки и накопительные задачи
  extended_curriculum.py  # 108 уроков расширенной части
  learning_design.py      # метаданные, интервалы и curriculum-linter

  conftest.py             # изолированная временная SQLite-база для тестов
docs/                     # карта, архитектура и библия обучения
```

## Безопасность учебного раннера

Встроенный редактор рассчитан на короткие учебные задачи. Разрешены безопасные модули стандартной библиотеки, async и виртуальные файлы/`pathlib`; доступ остаётся внутри временной папки. Системные, процессные, произвольные сетевые операции и небезопасные импорты запрещены, решение запускается отдельным процессом и останавливается по таймауту. Это защита для локального тренажёра, но **не** полноценный sandbox для публичного многоарендного сервиса. Перед публикацией в интернете нужен отдельный контейнерный раннер с лимитами CPU, памяти и сети. Детали — в [SECURITY.md](SECURITY.md).
