## Принципы сложности

- Сначала один новый концепт за урок, затем — его комбинация с уже знакомыми. Функции идут шагами: вызов → параметр → `return` → сборка → именованный аргумент.
- Условия, циклы и коллекции также разделены: `if` → `else`, `for` → накопитель, `while` → `break`, список → `append` → `split` → `sum`, словарь → множество.
- Curriculum-linter проверяет теорию, варианты, заготовки, подсказки и скрытые тесты: синтаксис, builtin или метод не может появиться до вводящего урока.
- Перед ответом ученик видит нейтральный план; подсказки открываются уровнями и только последняя приближается к готовой форме.
- Начальный блок повторяет базовые действия в разных контекстах, чтобы сформировать память до перехода к условиям и циклам.
- Поведенческие тесты принимают эквивалентные решения; структурная AST-проверка используется только когда форма прямо названа учебной целью.
- Урок требует обязательный самостоятельный код. Контрольная точка не открывается до завершения раздела и не сдаётся одними вариантами ответа.
- Новые навыки возвращаются в смешанной практике по интервалам 1/3/7/14/30; слабое место закрывается после двух верных повторений подряд.
- Проекты в конце повторяют все ключевые слои: данные, логику, I/O, API, тесты и документацию.

## Мастерская проектов

Параллельно с курсом открывается шесть небольших практических инструментов. Каждый проект требует предыдущий проект и конкретные lesson ID, поэтому не прыгает в незнакомые темы:

1. Именная открытка — `input()`, переменная и f-строка.
2. Калькулятор покупки — преобразование `int()` и арифметика.
3. Проверка доступа — условие `if` / `else`.
4. Счётчик слов — `split()`, список и `len()`.
5. Сводка оценок — список, `sum()` и среднее.
6. Карточка контакта — словарь и красивый вывод данных.

У каждого проекта есть стартовый код, контрольный список, постепенно раскрываемые подсказки, безопасный запуск с консольным выводом и несколько контрольных сценариев. Захардкоженный ответ под один пример проверку не проходит.

Подробные обязательные правила авторинга, повторения, экзаменов и проверки эквивалентных решений находятся в [LEARNING_DESIGN.md](LEARNING_DESIGN.md).
